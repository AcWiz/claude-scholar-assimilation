# 学术图表视觉规范

> 来源：awesome-ai-research-writing prompt 10 + claude-scholar visualization-best-practices

## 架构图风格要求

### 风格基调
- 顶会论文风格：专业、干净、现代、极简主义
- 扁平化矢量插画风格，线条简洁
- 参考 DeepMind 或 OpenAI 论文中的图表美学
- 拒绝卡通感、油画感或过度艺术化
- 背景必须是纯白色，无任何纹理或阴影

### 色彩体系
- 严格使用淡色系或柔和色调
- 严禁使用过于鲜艳饱和的颜色（如大红大绿）
- 严禁使用过于暗淡沉重的颜色
- 利用颜色深浅变化区分不同模块类型

### 布局原则
- 清晰的模块划分和数据流箭头
- 组织方式：左到右、上到下、循环等
- 相关组件逻辑分组
- 适当使用现代简洁的矢量图标

### 文字规范
- 所有文字必须使用英文
- 为关键模块和方程式添加清晰易读的文本标签
- 严禁长句子、描述性段落或复杂公式
- 文字用来说明模块身份，不用来解释原理

### 禁止事项
- 逼真照片感
- 杂乱的草图线条
- 难以辨认的文本
- 廉价的 3D 阴影

## 实验图表视觉规范

### 色盲友好配色

**Okabe-Ito 配色方案（首选）**：
| 颜色 | HEX 值 |
|------|--------|
| 橙色 | #E69F00 |
| 天蓝 | #56B4E9 |
| 绿色 | #009E73 |
| 黄色 | #F0E442 |
| 蓝色 | #0072B2 |
| 红色 | #D55E00 |
| 粉色 | #CC79A7 |
| 黑色 | #000000 |

**Paul Tol 配色**：适用于定性数据，提供亮色、柔和、对比多种方案。

### 配色原则
- 最多使用 5-7 种颜色
- 避免红绿组合（红绿色盲无法区分）
- 测试黑白打印效果
- 使用不同线型配合颜色（实线、虚线、点线、点划线）

### 格式要求

**矢量图 vs 位图**：
| 格式 | 使用场景 |
|------|----------|
| PDF/EPS | 图表、曲线、示意图（推荐） |
| SVG | 网页展示 |
| PNG >= 600 DPI | 照片、截图 |

### 尺寸规范

| 类型 | 宽度 | 高度 |
|------|------|------|
| 单栏图 | 3.5 in (9 cm) | 2-3 in |
| 双栏图 | 7 in (18 cm) | 3-5 in |

纵横比：4:3 或 16:9

### 字体规范
- 坐标轴标签：10-12 pt
- 刻度标签：8-10 pt
- 图例：8-10 pt
- Caption：9-10 pt
- 最小字体不低于 8 pt

### 线条规范
- 线宽：1.5-2.0 pt
- 标记点：4-6 pt
- 网格线透明度：0.3

### 误差表示
- 柱状图：垂直误差条
- 折线图：误差带（半透明阴影，alpha=0.2-0.3）
- 颜色与主线一致
- Caption 中必须说明误差类型（SD / SE / CI）

### 坐标轴规范
- Y 轴通常从 0 开始（除非有特殊原因说明）
- 不截断坐标轴夸大差异
- 使用科学计数法表示大数值
- 标签包含变量名和单位

### Caption 规范
- 独立完整，不依赖正文
- 说明图表内容、实验设置、关键观察
- 示例："Figure 1: Accuracy comparison across datasets. Error bars indicate standard deviation over 5 runs. Our method (blue) outperforms all baselines."
- 不在图内添加标题，使用 caption 代替
